Chapitre 1:Introduction

Les techniques de l’information et de la communication (TIC)

Les TIC regroupent des outils pour manipuler, traiter, transmettre et échanger des informations. Les TIC permettent une meilleure diffusion de l'information, de conserver et de présenter de l'information sous forme vocale, images ou des données contenues dans des signaux de nature optiques, électromagnétiques ou acoustiques. La naissance des TIC est due à la convergence de l'informatique, des télécommunications, des électroniques et de l'audiovisuel.

Chapitre 3: Le Modelé OSI

I. Les composants matériel d’un réseau informatique

Les Cartes Reseau

Permettent à un ordinateur de se connecter à un réseau, de transmettre de recevoir des donnes via des protocoles de communication spécifique, tel que TCP/IP, qui sont utilisés pour transférer des données sur internet.

Switch

Switch est un composant matériel qui connecte plusieurs appareils sur un réseau local (LAN) et leur permet de communiquer entre eux en utilisant des adresses MAC. Le switch stocke les adresses MAC dans une table de correspondance pour envoyer les donnes uniquement à l’appareil destinataire. Le switch sont disponibles en différentes taille et peuvent offrir des fonctionnalités avancées telles que la gestion des VLAN, la qualité de service QoS, la gestion du trafic, etc.

Hub

Hub est un composant matériel qui connecte plusieurs appareils sur un réseau local LAN. Contrairement à un switch le hub ne possède pas de table de correspondance pour envoyées sur le réseau les hubs ont été largement remplaces par des switches.

Le modèle OSI

Le modèle OSI est un modèle de référence pour les communications de réseau qui divise les communications en sept couche distinctes. Chaque couche est responsable de fonctions spécifique liées à la communication de données entre les appareils de réseau. Les sept couches sont la couche physique, la couche réseau, la couche session, la couche présentation et la couche application

Le Modele TCP/IP

Le modelé TCP/IP est un modèle de référence pour les communications de réseau qui divise les communications en 4 couches principales :
• La couche liaison de donnes
• La couche réseau
• La couche transport
• La couche application
Le modelé TCP IP est utilisé pour les plus largement utilises, tel que le protocole internet (IP), le protocole de contrôle de transmission (TCP), le protocole de datagramme utilisateur (UDP).

L'Adressage

Les communications dans un réseau obéissent a des règles telles que l’adressage, l’architecture et protocole une adresse IP internet protocole est le numéro qui identifie chaque ordinateur dans les réseaux qu’il appartient, il y a deux modes d’adressage : IPV4 et IPV6 L’adresse IPV4 contient une adresse source sur 32 bits et une adresse destinataire sur 32 bits, tandis que l’adresse IPV6 est codée sur 128 bits et est représentée en découpant le mot de 128 bits de l’adresse en 8 mot de 16 bits sépares par le caractère <<:>> chacun deux est en hexadécimal.

L'Adresse IPV4

L’adresse IPV4 est composée de 4 octettes divises en deux partie : une partie réseau et une partie hôte, chaque adresse IPV4 contient deux partie : une première partie des bits de valeur supérieure, l’adresse réseau, et une deuxième partie des bits, l’adresse hôte. On reconnait qu’un réseau est un hôte quand la valeur des bits pour la partie réseau est la même. Le nombre de bits contenus dans la partie hôtes détermine le nombre de machine possibles sur ce réseau. Les différents types d’adresse IPV4 sont l’adresse réseau ou tous les bits de la partie hôtes sont à 0, et l’adresse de diffusion qui est une adresse dans laquelle tous les bits la partie hôte sont à 1.

Chapitre 5:Système de Gestion de Version

Qu'est ce que Git ?

Git est logiciel de versioning créer en 2005 par Linus Torvalds , le créateur de Linux . Ce logiciel permet de :
•Conserver un historique des modifications effectuées sur un projet précédent
•Travailler en équipe de manière beaucoup plus efficace
•Facilitent grandement la gestion de projet

À quoi sert concrètement un système de Gestion de Version?

Le symbole de Gestion de Version permet d'établir un historique de toutes les modifications effectuées sur un élément faites par chaque développeur, pour ainsi avoir la possibilité de récupérer une version antérieure selon la date et l'heure da la sauvegarde. Chaque développeur pourrait également copier l'intégralité du contenu du serveur pour travailler en local sur sa machine.

Les deux modèles des logiciels de gestion de version : modèle centralisée VS modèle décentralisé

De manière plus simple pour installer Git c'est télécharger la dernière version sur le site officiel http://git-scm.com/downloads ; d'ouvrir le fichier téléchargé et de suivre les instructions à l'écran en laissant toute les valeurs par défaut .

Paramétrage de Git:

Une fois installé Git sur votre machine vous devez personnaliser votre environnement Git pour ne pas avoir à les fournir à nouveau plus tard . Pour faire cela, nous allons utiliser notre premier commande Git qui est la commande git config. Qui permet de configurer et modifier les variables des configurations de comportement de Git . Ensuite on passe l.option globale qui permet de personnaliser le comportement par défaut de certaines commandes . On va donc taper les commandes suivantes: gift.config global user .name et gift.config–globale user.mail.
